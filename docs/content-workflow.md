# Content Workflow

Use Markdown or MDX for most site content.

- Blog posts live in `src/content/blog/<slug>/index.md`.
- Project case studies live in `src/content/projects/<slug>/index.md`.
- Original Jupyter notebooks live in `notebooks/source/`.
- Converted notebook posts should be reviewed before publishing.

Keep project pages outcome-oriented: problem, data, method, result, limitations, and next steps.
